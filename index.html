Debug

Debug en NetBeans

NetBeans integra un depurador visual completo basado en JPDA. Permite pausar la ejecución en cualquier línea, inspeccionar variables y el estado de la JVM en tiempo real.

1
Añadir un breakpoint
Haz clic en el margen izquierdo de cualquier línea ejecutable. Aparecerá un cuadrado rojo. También puedes pulsar Ctrl+F8 con el cursor en esa línea.
2
Iniciar el depurador
Pulsa Ctrl+F5 o ve a Debug → Debug Project. La ejecución se pausará cuando alcance el primer breakpoint.
3
Navegar por el código
F8 avanza una línea (Step Over) · F7 entra en el método (Step Into) · Shift+F7 sale del método (Step Out) · F9 continúa hasta el siguiente breakpoint.
4
Inspeccionar variables
La pestaña Variables en el panel inferior muestra el valor de cada variable en el ámbito actual. Puedes expandir objetos para ver sus atributos.
5
Watches y evaluación de expresiones
En la pestaña Watches añade expresiones como lista.size() o obj.getNombre() para monitorizar su valor en cada paso.
💡Breakpoint condicional: clic derecho sobre el breakpoint → Breakpoint Properties → activa Condition y escribe, por ejemplo, i == 10. El programa solo se detendrá cuando se cumpla.
Javapublic class EjemploDebug { public static void main(String[] args) { int[] numeros = {3, 7, 0, 2}; int suma = 0; // ← Pon el breakpoint aquí para observar el bucle for (int n : numeros) { suma += n; // Observa "suma" en la pestaña Variables } System.out.println("Suma: " + suma); } }

Debug en VS Code

Requiere la extensión Extension Pack for Java de Microsoft. Una vez instalada, el flujo de debug es completo y visual, integrado en el panel lateral.

1
Instalar la extensión
Busca Extension Pack for Java en el Marketplace e instálala. Incluye el Language Server, debugger, soporte Maven/Gradle y más.
2
Añadir breakpoints
Haz clic en el margen izquierdo del editor (aparece un punto rojo) o pulsa F9 con el cursor en la línea.
3
Iniciar el debug
Pulsa F5. Si no hay launch.json, VS Code lo genera automáticamente detectando la clase con main().
4
Controles de navegación
F10 Step Over · F11 Step Into · Shift+F11 Step Out · F5 Continuar · Shift+F5 Detener.
5
Panel Run and Debug
El panel lateral Run and Debug (Ctrl+Shift+D) muestra variables locales, variables globales, pila de llamadas y breakpoints activos.
💡Clic derecho sobre una variable durante el debug → Add to Watch. También puedes hacer hover sobre cualquier variable para ver su valor inline en el editor.
JSON — launch.json{ "version": "0.2.0", "configurations": [ { "type": "java", "name": "Launch EjemploDebug", "request": "launch", "mainClass": "com.ejemplo.EjemploDebug", "args": [] } ] }

Debug en IntelliJ IDEA

IntelliJ IDEA tiene uno de los depuradores más potentes del mercado. Incluye evaluación de expresiones en caliente, Smart Step Into y modificación de valores en tiempo de ejecución.

1
Añadir breakpoints
Haz clic en el margen izquierdo (gutter). Los breakpoints se muestran con un círculo rojo. Ctrl+F8 alterna el breakpoint de la línea actual.
2
Iniciar el depurador
Pulsa Shift+F9 o el icono del insecto verde. También puedes hacer clic derecho en la clase → Debug 'NombreClase.main()'.
3
Navegar por el código
F8 Step Over · F7 Step Into · Shift+F8 Step Out · F9 Resume · Alt+F9 Run to cursor (sin breakpoint).
4
Evaluar expresiones
Pulsa Alt+F8 para abrir el evaluador de expresiones. Puedes ejecutar cualquier código Java en el contexto actual del programa. Muy útil para probar hipótesis.
5
Modificar valores en caliente
En el panel Variables, haz doble clic en el valor de una variable para modificarlo mientras el programa está pausado. Ideal para probar escenarios sin reiniciar.
💡Smart Step Into (Shift+F7): cuando una línea tiene varias llamadas encadenadas, IntelliJ te pregunta en cuál quieres entrar en lugar de entrar en todas. Enormemente útil con streams y lambdas.
⚠La función Hot Swap permite modificar el código fuente y recargarlo sin reiniciar el programa, pero solo funciona con cambios dentro de métodos. Cambios en la firma de métodos o atributos requieren reinicio.

Debug en Eclipse

Eclipse tiene un depurador maduro integrado en su perspectiva Debug. Cambia automáticamente de perspectiva al iniciar el debug, mostrando todos los paneles necesarios.

1
Añadir breakpoints
Doble clic en el margen izquierdo de cualquier línea o Ctrl+Shift+B. El breakpoint aparece como un punto azul.
2
Iniciar el depurador
Pulsa F11 o ve a Run → Debug As → Java Application. Eclipse te preguntará si quieres cambiar a la perspectiva Debug: acepta.
3
Navegar por el código
F6 Step Over · F5 Step Into · F7 Step Return · F8 Resume · Ctrl+R Run to Line.
4
Perspectiva Debug
La perspectiva muestra: panel Debug (pila de llamadas), Variables (valores actuales), Breakpoints (lista de puntos de ruptura) y Expressions (watches).
5
Display view y evaluación
Abre Window → Show View → Display. Escribe cualquier expresión Java, selecciónala y pulsa Ctrl+Shift+D para evaluarla en el contexto actual.
💡Breakpoint condicional: clic derecho en el breakpoint → Breakpoint Properties → activa Enable Condition. También puedes configurar que se detenga solo después de N pasos (Hit Count).
Unit Test

Unit Test en NetBeans (JUnit)

NetBeans tiene soporte nativo para JUnit 4 y JUnit 5 y puede generar automáticamente el esqueleto de la clase de test.

1
Generar la clase de test
Clic derecho en la clase → Tools → Create/Update Tests. NetBeans crea la clase en test/ con un método por cada método público.
2
Escribir los métodos de test
Decora cada método con @Test y usa assertEquals, assertTrue, assertThrows, etc. del paquete org.junit.jupiter.api.
3
Ejecutar los tests
Alt+F6 ejecuta todos los tests del proyecto. Clic derecho en la clase de test → Run File ejecuta solo esa clase.
4
Interpretar resultados
El panel Test Results muestra en verde los tests pasados y en rojo los fallidos, con el mensaje de error y la traza de pila exacta.
Java — JUnit 5import org.junit.jupiter.api.*; import static org.junit.jupiter.api.Assertions.*; class CalculadoraTest { private Calculadora calc; @BeforeEach void setUp() { calc = new Calculadora(); } @Test void testSuma() { assertEquals(7, calc.suma(3, 4), "3+4 debe ser 7"); } @Test void testDivisionPorCero() { assertThrows(ArithmeticException.class, () -> calc.dividir(10, 0)); } }
⚠Para proyectos Maven añade JUnit al pom.xml con <scope>test</scope>. Para proyectos Ant, añade el JAR de JUnit en Project Properties → Libraries → Add JAR/Folder.
Unit Test

Unit Test en VS Code (JUnit)

Con el Extension Pack for Java, VS Code detecta automáticamente las clases de test y muestra un panel de Testing completo con ejecución individual o masiva.

1
Configurar JUnit en Maven
Añade la dependencia en el pom.xml. VS Code detectará automáticamente el framework.
2
Crear la clase de test
Crea manualmente la clase en src/test/java/. VS Code la reconoce por las anotaciones @Test.
3
Panel Testing
Abre el panel Testing (icono de tubo de ensayo en la barra lateral). Muestra todos los tests organizados en árbol.
4
Ejecutar y depurar tests
Junto a cada método @Test aparece un icono de play. Puedes ejecutar o depurar un test individual sin salir del editor.
XML — pom.xml<dependency> <groupId>org.junit.jupiter</groupId> <artifactId>junit-jupiter</artifactId> <version>5.10.2</version> <scope>test</scope> </dependency>
💡VS Code muestra el resultado inline junto al método: tick verde o X roja. Con @ParameterizedTest y @ValueSource puedes ejecutar el mismo test con múltiples valores de entrada.
Unit Test

Unit Test en IntelliJ IDEA (JUnit)

IntelliJ IDEA ofrece la experiencia de testing más completa: genera tests automáticamente, soporta JUnit 4/5 y TestNG, e integra cobertura de código (Coverage) de serie.

1
Generar la clase de test
Coloca el cursor en el nombre de la clase → Alt+Enter → Create Test. Un asistente te permite elegir el framework y marcar qué métodos testear.
2
Escribir los tests
IntelliJ añade JUnit automáticamente a las dependencias del proyecto. Escribe los métodos con @Test y los assertions de JUnit 5.
3
Ejecutar tests
Ctrl+Shift+F10 ejecuta el fichero actual. El panel inferior muestra los resultados en árbol con tiempo de ejecución.
4
Cobertura de código
Clic derecho → Run with Coverage. IntelliJ colorea el código en verde (cubierto) y rojo (no cubierto), y muestra el porcentaje por clase y método.
💡Con Alt+Enter en un assertion fallido, IntelliJ sugiere corregir el valor esperado al valor real. También genera automáticamente los imports necesarios.
ℹIntelliJ IDEA Community (gratuita) incluye soporte completo de JUnit. La edición Ultimate añade integración con frameworks de testing adicionales como Mockito, Spock o Cucumber.
Unit Test

Unit Test en Eclipse (JUnit)

Eclipse fue uno de los primeros IDEs en integrar JUnit nativamente. La integración es madura y completa, con soporte para JUnit 4 y JUnit 5 mediante el launcher Jupiter.

1
Añadir JUnit al proyecto
Clic derecho en el proyecto → Build Path → Add Libraries → JUnit. Elige JUnit 5 (Jupiter) para proyectos nuevos.
2
Generar clase de test
Clic derecho en la clase → New → JUnit Test Case. Eclipse genera el fichero en una carpeta test/ separada.
3
Ejecutar tests
Clic derecho en la clase de test → Run As → JUnit Test o Alt+Shift+X, T. La vista JUnit se abre automáticamente.
4
Vista JUnit
Muestra la barra de progreso (verde/rojo), el árbol de tests con su estado, y el mensaje de error completo al hacer clic en un test fallido.
💡Eclipse integra EclEmma para medir la cobertura: clic derecho → Coverage As → JUnit Test. El código cubierto aparece en verde, el no cubierto en rojo, directamente en el editor.
Refactor

Refactorización en NetBeans

NetBeans realiza refactorizaciones automáticas y seguras, actualizando todas las referencias en el proyecto antes de aplicar el cambio, con vista previa.

1
Renombrar (Rename)
Clic derecho sobre variable, método o clase → Refactor → Rename (Ctrl+R). Actualiza todas las referencias automáticamente.
2
Extraer método
Selecciona un bloque de código → Refactor → Extract Method. NetBeans detecta los parámetros necesarios y crea el método nuevo.
3
Mover clase
Clic derecho en la clase → Refactor → Move. Mueve la clase a otro paquete y actualiza todos los imports del proyecto.
4
Encapsular campos
Clic derecho en un atributo → Refactor → Encapsulate Fields. Genera getters y setters y reemplaza el acceso directo por los métodos.
5
Vista previa de cambios
Antes de aplicar cualquier refactorización, NetBeans muestra un panel con todos los cambios que realizará. Puedes desmarcar los que no desees.
💡Usa Refactor → Find Usages antes de refactorizar para ver todos los lugares del código que se verán afectados. Evita sorpresas en clases que no conoces bien.
Refactor

Refactorización en VS Code

VS Code ofrece refactorizaciones a través del menú contextual y el bombillo de sugerencias (Ctrl+.). Con la extensión Java activa, cubre los casos más comunes.

1
Renombrar símbolo
Clic derecho → Rename Symbol (F2). Renombra en todos los ficheros del proyecto, incluyendo referencias en otras clases.
2
Extraer método o variable
Selecciona el bloque → clic derecho → Refactor... → Extract Method o Extract to constant/field.
3
Sugerencias del bombillo
El icono de bombillo amarillo (o Ctrl+.) sugiere refactorizaciones según el contexto del cursor: convertir a lambda, invertir condición, etc.
4
Generar código
Clic derecho → Source Action para generar getters/setters, constructores, toString(), equals() y hashCode() automáticamente.
⚠VS Code tiene menos opciones de refactorización que NetBeans o IntelliJ. Para refactorizaciones avanzadas (mover clase, extraer interfaz, inline method) considera complementarlo con la extensión Java Code Generators.
Refactor

Refactorización en IntelliJ IDEA

IntelliJ IDEA tiene el sistema de refactorización más completo de los cuatro IDEs, con soporte para más de 40 refactorizaciones automáticas y análisis de impacto antes de aplicar.

1
Menú Refactor
Clic derecho → Refactor o Ctrl+Alt+Shift+T abre el menú de refactorizaciones disponibles para el elemento bajo el cursor.
2
Rename
Shift+F6. Soporta renombrar con preview, busca incluso en comentarios y cadenas de texto. Resalta las referencias antes de confirmar.
3
Extract / Inline
Extract Method (Ctrl+Alt+M), Extract Variable (Ctrl+Alt+V), Extract Constant (Ctrl+Alt+C), Inline (Ctrl+Alt+N).
4
Refactorizaciones de diseño
Extract Interface/Superclass, Pull Members Up/Down, Move (F6), Safe Delete, Change Signature (Ctrl+F6).
5
Inspecciones y correcciones
Alt+Enter sobre cualquier advertencia muestra el menú de corrección rápida. IntelliJ detecta code smells y propone la refactorización adecuada.
💡Safe Delete: IntelliJ comprueba si el elemento tiene referencias antes de borrarlo. Si las tiene, muestra un aviso con todos los usos. Nunca borrarás código usado por accidente.
Refactor

Refactorización en Eclipse

Eclipse ofrece un conjunto sólido de refactorizaciones automáticas accesibles desde el menú Refactor o mediante atajos de teclado.

1
Rename
Selecciona el identificador → Alt+Shift+R. Eclipse renombra en todos los ficheros y muestra una vista previa de los cambios.
2
Extract Method
Selecciona el bloque → Alt+Shift+M. Eclipse infiere automáticamente los parámetros y el tipo de retorno del nuevo método.
3
Extract Local Variable / Constant
Alt+Shift+L extrae una expresión a variable local. Útil para dar nombre semántico a expresiones complejas.
4
Move y Change Method Signature
Alt+Shift+V mueve la clase. Alt+Shift+C cambia la firma de un método actualizando todas sus llamadas.
5
Quick Fix
Ctrl+1 sobre cualquier error o advertencia muestra las correcciones disponibles, muchas de ellas refactorizaciones automáticas.
💡El menú Source de Eclipse complementa a Refactor: genera constructores, getters/setters, toString(), organiza imports (Ctrl+Shift+O) y formatea código (Ctrl+Shift+F).
Javadoc

Generar Javadoc en NetBeans

NetBeans genera la documentación HTML desde los comentarios /** ... */ del código con un solo clic, sin necesidad de usar el terminal.

1
Escribir los comentarios Javadoc
Escribe /** encima de una clase o método y pulsa Enter. NetBeans autocompleta la estructura con los @param y @return correctos.
2
Usar las etiquetas estándar
Completa @param nombre descripción, @return descripción, @throws ExcepcionClass motivo, @author nombre, @version 1.0.
3
Generar el Javadoc
Ve a Run → Generate Javadoc. NetBeans invoca la herramienta javadoc y guarda el HTML en dist/javadoc/.
4
Visualizar la documentación
El Javadoc se abre automáticamente en el navegador. El fichero principal es dist/javadoc/index.html.
Java — Javadoc/** * Representa una calculadora básica de operaciones aritméticas. * * @author Ana García * @version 1.2 * @since 2024 */ public class Calculadora { /** * Divide dos enteros. * * @param dividendo el número a dividir * @param divisor el divisor (no puede ser cero) * @return el resultado de la división entera * @throws ArithmeticException si divisor es 0 */ public int dividir(int dividendo, int divisor) { if (divisor == 0) throw new ArithmeticException("División por cero"); return dividendo / divisor; } }
💡Activa la previsualización del Javadoc mientras escribes: abre el panel Javadoc en la ventana de autocompletado (Ctrl+Space), o coloca el cursor sobre cualquier clase/método de tu código.
Javadoc

Generar Javadoc en VS Code

VS Code no genera Javadoc de forma nativa, pero con la extensión adecuada para autocompletar los comentarios y el terminal integrado, el proceso es rápido.

1
Instalar extensión de ayuda
Instala Javadoc Tools desde el Marketplace. Añade soporte para generar el javadoc directamente desde VS Code y autocompleta los comentarios.
2
Autocompletar comentarios
Escribe /** encima de un método y pulsa Enter. La extensión genera automáticamente los @param y @return.
3
Generar desde el terminal
Abre el terminal integrado (Ctrl+`) y ejecuta el comando javadoc indicando el paquete fuente y el directorio de salida.
4
Abrir el resultado
Abre docs/index.html en el navegador. Con la extensión Live Preview puedes verlo directamente dentro de VS Code.
Bash — Terminal VS Code# Generar Javadoc para un paquete completo javadoc -d docs -sourcepath src -subpackages com.miapp \ -author -version -encoding UTF-8 # Para un solo fichero javadoc -d docs src/Calculadora.java -author -version # Con proyecto Maven (alternativa) mvn javadoc:javadoc # Resultado en: target/site/apidocs/index.html
Javadoc

Generar Javadoc en IntelliJ IDEA

IntelliJ IDEA tiene soporte excelente para Javadoc: autocompleta las etiquetas, muestra la documentación renderizada en el popup de ayuda y genera el HTML desde el menú.

1
Autocompletar comentarios Javadoc
Escribe /** sobre un método o clase y pulsa Enter. IntelliJ genera la plantilla completa con todos los parámetros y el retorno.
2
Previsualizar documentación
Ctrl+Q sobre cualquier elemento muestra su Javadoc renderizado. Muy útil para verificar que la documentación tiene sentido antes de generar el HTML.
3
Generar el HTML
Ve a Tools → Generate JavaDoc. Un diálogo te permite elegir el ámbito (proyecto completo, módulo o paquete), el directorio de salida y opciones adicionales.
4
Parámetros adicionales
En el campo Other command line arguments del diálogo añade -author -version -encoding UTF-8 para incluir esas etiquetas en la documentación generada.
💡IntelliJ avisa con un warning si un método público no tiene comentario Javadoc. Activa la inspección en Settings → Editor → Inspections → Javadoc para localizar fácilmente la documentación pendiente.
Javadoc

Generar Javadoc en Eclipse

Eclipse integra un asistente gráfico para generar Javadoc que permite configurar el ámbito, la visibilidad y los parámetros del comando en un único diálogo.

1
Autocompletar comentarios
Escribe /** encima de una clase o método y pulsa Enter. Eclipse genera la plantilla con las etiquetas correctas.
2
Abrir el asistente Javadoc
Ve a Project → Generate Javadoc. Eclipse abre un asistente de tres pasos.
3
Configurar el asistente
Paso 1: selecciona el proyecto y la visibilidad (public/protected). Paso 2: indica el directorio de destino. Paso 3: añade argumentos extra como -author -version.
4
Generar y abrir
Haz clic en Finish. Eclipse genera el HTML en la carpeta indicada y puedes abrirlo en el navegador externo o en el visor interno de Eclipse.
💡Eclipse muestra el Javadoc de cualquier clase o método en la vista Javadoc (Window → Show View → Javadoc). Al mover el cursor, se actualiza automáticamente mostrando la documentación del elemento actual.
Crear .jar

Crear un .jar en NetBeans

NetBeans genera el .jar ejecutable automáticamente al compilar el proyecto, incluyendo el MANIFEST.MF con la clase principal y el classpath de librerías.

1
Configurar la clase principal
Clic derecho en el proyecto → Properties → Run. En el campo Main Class, indica la clase que contiene el método main().
2
Compilar y empaquetar
Pulsa F11 (Build Project). NetBeans compila, ejecuta los tests y genera el .jar en la carpeta dist/.
3
Localizar el artefacto
El archivo se genera en dist/NombreProyecto.jar. Si hay librerías externas, se copian en dist/lib/ y se referencian en el MANIFEST.MF.
4
Verificar la ejecución
Abre un terminal en la carpeta del proyecto y ejecuta java -jar dist/NombreProyecto.jar para confirmar que funciona correctamente.
MANIFEST.MF — generado por NetBeansManifest-Version: 1.0 Ant-Version: Apache Ant 1.10.x Main-Class: com.ejemplo.Main Class-Path: lib/commons-lang3.jar lib/gson.jar # Ejecutar desde el terminal java -jar dist/MiProyecto.jar java -jar dist/MiProyecto.jar argumento1 argumento2
💡Para crear un fat jar (con todas las librerías incluidas dentro del propio JAR), necesitas configurar la tarea Ant en build.xml o usar el plugin Maven Shade si el proyecto usa Maven.
Crear .jar

Crear un .jar en VS Code

En VS Code hay tres formas de crear el .jar: usando el comando de exportar de la extensión Java, con Maven desde el terminal, o con los comandos básicos del JDK.

1
Opción A — Export Jar (extensión)
Abre la paleta de comandos con Ctrl+Shift+P → escribe Java: Export Jar. Un asistente guiado te pide la clase principal y el destino del .jar.
2
Opción B — Maven Package
Si el proyecto usa Maven, ejecuta mvn package en el terminal integrado. El .jar aparece en target/NombreProyecto-version.jar.
3
Opción C — Comandos javac y jar
Compila manualmente con javac y empaqueta con el comando jar. Útil para proyectos sin herramienta de build o para entender el proceso.
4
Verificar el contenido
Usa jar tf NombreProyecto.jar para listar los ficheros incluidos y verificar que el MANIFEST.MF tiene la clase principal correcta.
Bash# ── Opción Maven ────────────────────────────── mvn package java -jar target/MiProyecto-1.0-SNAPSHOT.jar # ── Opción manual (sin Maven) ───────────────── # 1. Compilar todos los .java javac -d out src/com/ejemplo/*.java # 2. Crear el JAR con clase principal jar cfe MiProyecto.jar com.ejemplo.Main -C out . # 3. Verificar contenido jar tf MiProyecto.jar
⚠Sin Maven o Gradle, debes gestionar el classpath manualmente con el flag -cp si el proyecto usa librerías externas. Para proyectos reales se recomienda siempre usar un gestor de dependencias.
Crear .jar

Crear un .jar en IntelliJ IDEA

IntelliJ IDEA permite crear artefactos JAR desde su sistema de artefactos, con soporte para incluir dependencias y configurar el MANIFEST.MF de forma visual.

1
Abrir la configuración de artefactos
Ve a File → Project Structure (Ctrl+Alt+Shift+S) → sección Artifacts → botón + → JAR → From modules with dependencies.
2
Configurar el artefacto
Selecciona el módulo y la Main Class. Elige si extraer las dependencias dentro del JAR (fat jar) o referenciarlas externamente. Haz clic en OK.
3
Compilar el artefacto
Ve a Build → Build Artifacts → selecciona el artefacto → Build. El .jar se genera en la carpeta out/artifacts/.
4
Alternativa con Maven
Si el proyecto usa Maven, usa el panel Maven en la barra lateral → Lifecycle → package. O configura el plugin maven-jar-plugin en el pom.xml.
XML — pom.xml con maven-jar-plugin<plugin> <groupId>org.apache.maven.plugins</groupId> <artifactId>maven-jar-plugin</artifactId> <version>3.3.0</version> <configuration> <archive> <manifest> <mainClass>com.ejemplo.Main</mainClass> </manifest> </archive> </configuration> </plugin>
💡Para un fat jar (todas las dependencias incluidas), usa el plugin maven-shade-plugin o maven-assembly-plugin en lugar de maven-jar-plugin. Esto genera un JAR completamente autocontenido.
Crear .jar

Crear un .jar en Eclipse

Eclipse incluye un asistente JAR Export que guía paso a paso la creación del archivo, con opciones para incluir fuentes, recursos y configurar el MANIFEST.MF.

1
Abrir el asistente JAR Export
Clic derecho en el proyecto → Export → Java → JAR file → Next.
2
Seleccionar recursos
Marca el proyecto o los paquetes que quieres incluir. Puedes incluir o excluir los fuentes (.java) y los recursos (.properties, imágenes, etc.).
3
Destino del archivo
Indica la ruta completa donde se guardará el .jar y haz clic en Next.
4
Configurar MANIFEST.MF
En el último paso selecciona Generate the manifest file e indica la Main class. También puedes usar un MANIFEST.MF propio si ya lo tienes preparado.
5
Finalizar y verificar
Haz clic en Finish. Verifica con java -jar NombreProyecto.jar desde el terminal que el programa arranca correctamente.
⚠El asistente JAR de Eclipse no incluye las librerías externas (.jar de terceros) en el artefacto generado. Para incluirlas debes usar el asistente Runnable JAR file (en lugar de JAR file), que sí las empaqueta dentro.
💡Para proyectos Maven en Eclipse (con M2E), puedes hacer clic derecho → Run As → Maven build → escribe package como goal. Más rápido y reproducible que el asistente gráfico.
